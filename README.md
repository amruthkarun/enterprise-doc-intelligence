#  Enterprise Document Intelligence Platform

A Python-based web application for secure, multi-user document search with **role-based access control** and **conversational Q\&A**. This solution enables teams to log in, search through their authorized corporate documents (PDFs), and retrieve actionable insights using semantic search powered by advanced embeddings.

---

## Features

*  **Multi-user login with document-level access control**
*  **Semantic search** using `BAAI/bge-base-en-v1.5` embeddings (109M, rank-aware, CPU-friendly)
*  **Conversational Q\&A** with chat history maintained per user
*  **ChromaDB** vector store for fast and efficient retrieval
*  **Fully offline and runs on CPU** (No OpenAI required)
*  **Simple UI** with Streamlit

---

##  Tech Stack

| Component | Technology | Purpose |
|-----------|-----------|----------|
| **Frontend** | Streamlit | Web UI and user interaction |
| **Backend** | Python 3.12+ | Core application logic |
| **LLM Framework** | LangChain 1.3.11+ | Document processing & RAG |
| **Vector Database** | ChromaDB 0.4.2+ | Persistent semantic search index |
| **Embeddings** | BAAI/bge-base-en-v1.5 (109M) | Semantic text representation |
| **PDF Parsing** | PyPDF / PyMuPDF | Document text extraction |
| **Vector Search** | FAISS (cpu) | Similarity matching |
| **Authentication** | Email-based | User access control |

**Key Dependencies:**
- `langchain==1.3.11`, `langchain-chroma==1.1.0`, `langchain-text-splitters==1.1.2`
- `sentence-transformers` for embeddings
- `torchvision` (required for embedding models)

##  Setup Instructions

### 1️ Clone the Repository

```bash
git clone https://github.com/amruthkarun/enterprise-doc-intelligence.git
cd enterprise-doc-intelligence
```

---

### 2️ Create and Activate a Conda Environment

```bash
# Create isolated environment with Python 3.12
conda create -n doc-search python=3.12 -y

# Activate the environment
conda activate doc-search
```

---

### 3️ Install Dependencies

```bash
# Install all required packages with pinned versions
pip install -r requirements.txt

# Verify installation (optional)
python -c "import langchain; import chromadb; print('✓ Installation successful')"
```

---

### 4️ Add Your PDF Files

Place your PDF files inside the `/data` folder.

Example filenames:

```
Allianz_Earnings_Full_Transcript.pdf
Apple_Earnings_Full_Transcript.pdf
Corporate_Earnings_Transcripts_2026.pdf
JIO_Earnings_Full_Transcript.pdf
```


### 5️ Build the Vector Index

```bash
cd src
python build_index.py
```

This script will:

* Parse PDFs
* Split text into chunks
* Generate embeddings with `BAAI/bge-base-en-v1.5`
* Save them into `chroma_db/`

---

### 6️ Run the Streamlit App

```bash
streamlit run app.py
```

Open the browser at:

```
http://localhost:8501
```

##  User Access Control & Permissions

Edit `src/user_auth.py` to define role-based document access:

```python
USER_PERMISSIONS = {
    "alice@email.com": ["Allianz_Earnings_Full_Transcript"],
    "bob@email.com": ["Apple_Earnings_Full_Transcript", "Corporate_Earnings_Transcripts_2026"],
    "charlie@email.com": ["JIO_Earnings_Full_Transcript"],
    "admin@email.com": ["Allianz_Earnings_Full_Transcript", "Apple_Earnings_Full_Transcript", 
                        "Corporate_Earnings_Transcripts_2026", "JIO_Earnings_Full_Transcript"]
}

def get_user_access(email):
    """Return list of accessible documents for user email."""
    return USER_PERMISSIONS.get(email, [])
```

**Important Notes:**
- Document names must exactly match PDF filenames (WITHOUT `.pdf` extension)
- Users not in `USER_PERMISSIONS` receive "Access Denied" message
- Access is enforced on every search query (real-time)

##  Embedding Model Details

**Current Model:** [`BAAI/bge-base-en-v1.5`](https://huggingface.co/BAAI/bge-base-en-v1.5)

**Model Specifications:**
- **Parameters:** 109M
- **Size:** ~220MB (disk) / ~400MB (loaded)
- **Embedding Dimension:** 768
- **Language:** English and code
- **Query Latency:** ~50-100ms per query (CPU)

**Why This Model:**
- ✅ Optimized for semantic search (rank-aware)
- ✅ Higher quality embeddings for domain-specific content
- ✅ Better recall and precision for corporate documents
- ✅ Fully reproducible and open source
- ✅ Offline operation after first download

**Alternative Models (if needed):**
- `sentence-transformers/all-MiniLM-L6-v2` (22M, lightweight for resource-constrained systems)
- `sentence-transformers/all-mpnet-base-v2` (109M, alternative quality model)

##  Troubleshooting

### "No relevant information found"
- Run `python build_index.py` to build vector index
- Verify document names in `user_auth.py` match PDF filenames (no `.pdf`)
- Check PDFs exist in `data/` and contain extractable text

### "ModuleNotFoundError" for langchain packages
- Reinstall: `pip install --upgrade -r requirements.txt`
- Verify Python 3.12+: `python --version`
- Clear cache: `pip cache purge`

### Streamlit port 8501 already in use
- Use alternative: `streamlit run app.py --server.port 8502`
- Or: `netstat -an | findstr 8501` to identify process

### Slow queries or out of memory
- First run takes 1-2 minutes (model loading)
- Subsequent queries: 50-200ms
- Close other applications to free RAM
